# ASEI Output and State (`output/`)

This directory is the primary storage for the persistence and state tracking layers of the ASEI system. It ensures that long-running processes (like the 8-stage pipeline or autonomous agent cycles) can be resumed and audited.

## Directory Structure

*   **`asei_state/`**: Stores the execution state of the Agentic Layer.
*   **`checkpoints/`**: Stores the intermediate data and progress of the Ingestion Pipeline.

---

## 1. ASEI State (`output/asei_state/`)

This folder contains JSON files generated by the `Orchestrator` after every agent completes its task.

*   **File Naming:** `orchestrator_YYYYMMDDTHHMMSSZ.json`
*   **Content:**
    *   `cycle_id`: Unique identifier for the autonomous cycle.
    *   `status`: `running`, `complete`, or `failed`.
    *   `current_agent`: The agent currently executing (used for recovery).
    *   **Reports**: Detailed output from each agent (Evolution, Conflict, Synthesis, etc.), including counts of flagged nodes, detected conflicts, and proposed hypotheses.
    *   `errors`: A list of any exceptions or failures encountered during the cycle.

---

## 2. Checkpoints (`output/checkpoints/`)

This folder is used by the `CheckpointManager` during the Ingestion Pipeline run.

*   **File Types:**
    *   `stage_XX_NAME.json`: The actual data extracted or processed during stage XX.
    *   `stage_XX_NAME.done`: An empty marker file indicating that the stage completed successfully.
*   **Utility:**
    *   Allows the pipeline to skip already completed stages using the `--from-stage` or default resume behavior.
    *   Provides a way to inspect the output of a specific stage (e.g., Stage 4 Chunking) without querying the database.

---

## Maintenance and Cleanup

*   **State Files**: These files can accumulate over time if the system is run in `--serve` mode. It is safe to archive or delete older `orchestrator_*.json` files once the cycles are complete and audited.
*   **Checkpoints**: Deleting these will force the pipeline to re-run from Stage 1 unless `--from-stage` is used with new data. Always clear these (`--fresh`) if the `PIPELINE_VERSION` in `settings.py` is changed.
